# 游戏设计文档：《极速打包专家》 (Perfect Pack)

## 1. 核心体验与设计目标
- **核心爽点**：满足人类对“秩序”、“收纳”和“消除缝隙”的本能渴望（强迫症福音）。
- **优越感来源**：空间利用的最大化。通过不断将不规则物品严丝合缝地塞满箱子，获得“收纳大师”的掌控感。
- **视觉/听觉反馈（Juice）**：
  - 物品吸附入格的“咔哒”声（清脆、有质感）。
  - 100% 填满时的爆屏金光、印章盖下的“PERFECT”提示音，以及箱子被快速推走的视觉特效。

## 2. 核心玩法机制

### 2.1 网格与箱子系统
- 游戏基于二维网格（Grid）系统（例如：5x5, 6x6, 8x6 的箱子空间）。
- 每个“箱子”（Level）都有其内部容量（总格数）。
- 玩家的目标是将输送带上的物品**拖拽并放入箱子**，且物品之间不能重叠，也不能超出箱子边界。

### 2.2 物品系统 (Items / Polyominoes)
- 物品由不同的网格块组成（类似俄罗斯方块的形状：L型、T型、方形、一字型，以及不规则多边形）。
- 物品以直观的生活物件展示（例如：一把L型的折叠伞、2x2方块的便当盒、1x3的键盘等）。
- **旋转**：玩家可以点击物品使其旋转 90 度，以适应不同的空间。

### 2.3 核心循环与节奏
- 游戏由多个短促的“订单（箱子）”组成。
- **传送带设定**：屏幕下方有一个不断向左滚动的传送带，上面出现当前箱子需要的物品。玩家必须在物品掉落进焚化炉/回收站前将其放入箱子。
- **时间与容错**：游戏的主要压力来自“传送带速度”和“思考空间拼图的耗时”。
- **过关判定**：
  - **Perfect Pack**：箱子被 100% 填满，获得最高分和特殊特效。
  - **Good Pack**：箱子填满率超过 80% 且没有更多适用物品，玩家可点击“强行发货”，获得普通分数。
  - **失败**：传送带上的关键物品掉落且箱子未达到及格线，或者时间耗尽。

## 3. 计分与奖励机制 (Risk & Reward)
- **连击系统 (Combo)**：连续达成 100% 满箱（Perfect Pack）会增加分数倍率（x2, x3, x4...）。一旦强行发货或失败，连击中断。
- **速度奖励**：发货越快，获得的时间奖励分数越高。
- **优越感反馈**：
  - 达到高连击时，游戏 BGM 会增加乐器音轨，节奏感增强。
  - 最终结算时评定段位（例如：D级“乱塞新手” -> S级“空间魔术师” -> SSS级“薛定谔的旅行箱”）。

## 4. 技术与实现架构设想

### 4.1 前端栈
- **框架**：React + Vite (与月球能源危机保持一致)。
- **交互**：使用 `framer-motion` 处理平滑的拖拽、放置动画和发货动画。可以使用原生的 Pointer Events 处理网格吸附。
- **状态管理**：简单的 React State / Redux (视复杂度而定)，维护：
  - `grid`: 二维数组表示箱子的占用状态。
  - `conveyorItems`: 传送带上的物品队列。
  - `score`, `combo`, `timeLeft` 等。

## 5. UI/UX 布局规划
- **顶部 HUD**：倒计时进度条、当前分数、Combo 倍率、当前段位。
- **中央区域**：一个清晰的网格箱子（带有虚线格辅助线）。当拖拽物品悬停时，提供阴影（绿色表示可放置，红色表示冲突）。
- **底部区域**：滚动的传送带，新物品从右侧出现，向左移动。
- **动作反馈**：放置成功时的微小屏幕震动（Haptics）和尘土飞扬的小粒子特效。
